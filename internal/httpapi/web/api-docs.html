Browser sessions

Use your existing Mesh login

The interactive console sends same-origin requests with your current hardened session cookie. It never asks for or stores an administrator bearer credential.

Unsafe requests

CSRF is added automatically

For POST, PUT, PATCH, and DELETE, the console reads the same-origin CSRF cookie and sends its value in X-Mesh-CSRF. The browser supplies the exact Origin.

Sensitive credentials

Agent and recovery calls stay disabled

Examples use placeholders only. Device credentials, enrollment tokens, recovery codes, and legacy bearer tokens should be used by their intended client—not pasted into this page.
