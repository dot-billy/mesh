# Product completion roadmap

The objective is a secure, approachable web application that manages the complete Nebula network and node lifecycle and proves a working private network from empty hosts in under five minutes. The current repository delivers the control-plane and managed-agent core, but it is not yet the packaged production product.

## 1. Secure network bootstrap — working

- Dedicated encrypted Nebula CA and artifact-signing key per network.
- Network and node inventory with overlap checks and safe IPv4 allocation.
- A continuous three-step empty-fleet onboarding path: a successful network create immediately opens a role-locked first-lighthouse form with explicit site, failure-domain, and public-UDP context, while **Finish later** preserves the new network. Saving that credential advances to a role-locked first member; saving the member credential opens the network's fail-closed deployment readiness. Every existing network card now also carries a resumable four-stage setup guide computed only from the already validated authenticated fleet snapshot: network creation, an operational lighthouse, an operational member, and the configured operational-lighthouse floor. It deterministically selects add, resume, diagnose, redundancy, or readiness as the next action; focuses an exact pending row without invalidating its saved credential; excludes revoked and enrolled-but-nonoperational nodes from completion; and labels peer reachability unproved. The enrollment disclosure may show a strictly validated bootstrap-handoff courier URL, but labels it unauthenticated and never receives the independent expected digest. Readiness uses one bounded 10-second timeout after each completed check, cancels it when closed or hidden, resumes when visible, and stops on sign-out so host activation can converge in place without overlapping polling. Its redundancy remediation opens a role-locked second-lighthouse form, asks for an independent failure domain and endpoint, and returns the saved credential to refreshed readiness. Every transition scrubs its token, handoff URL, and install/enroll/activate commands from the DOM. The real browser-to-packet gate requires the full path, proves the scheduler lifecycle, proves the pending backup does not count as active redundancy, and still reaches authenticated overlay traffic in four seconds.
- Lighthouse/member enrollment with node-local private-key generation.
- One-time enrollment, crash-safe recovery, valid Nebula certificates, deterministic configs, and audit events.
- Authenticated dashboard and admin CLI, with persistent opaque browser sessions, hybrid OIDC provisioning, and inventory-gated OIDC-only one-use recovery available for production HTTPS deployments.
- Server-enforced RBAC with `viewer`, `operator`, and `admin` roles; canonical OIDC subject, verified-email, and group bindings; policy-fingerprint session invalidation; explicit network-read, routine-write, trust/security, identity-management, and audit permissions; legacy/recovery compatibility; and dashboard action gating from the authenticated session projection.
